## Project at a glance

- **Project**: `commitizen` ‚Äî Python CLI for enforcing Conventional Commits, automating version bumps, and generating changelogs.
- **Library + CLI**: code is reachable both via `cz` and `import commitizen`.
- **Cross-platform**: tests run on Linux/macOS/Windows √ó Python 3.10‚Äì3.14. Avoid POSIX-only assumptions (paths, subprocesses, line endings).
- **Key entrypoints**:
    - `commitizen/cli.py` ‚Äî CLI definition (decli + argparse).
    - `commitizen/commands/` ‚Äî one module per `cz` subcommand.
    - `commitizen/config/` ‚Äî configuration discovery and parsing.
    - `commitizen/providers/` ‚Äî version providers.
    - `commitizen/changelog_formats/` ‚Äî changelog file formats.

## Do not touch

These files are regenerated by Commitizen-specific tooling, so hand-edits get reverted on the next release or doc rebuild:

- `CHANGELOG.md` ‚Äî produced by `cz changelog`. Hand-edits will be overwritten on the next release.
- `commitizen/__version__.py` ‚Äî bumped by `cz bump` via the configured version provider.
- `.pre-commit-config.yaml:rev:` lines under the `Commitizen` repo ‚Äî bumped by `cz bump` (`version_files` in `pyproject.toml`).
- `docs/images/cli_help/*.svg` and `docs/images/cli_interactive/*.gif` ‚Äî regenerated by `uv run poe doc:screenshots`. See the [update-snapshots playbook](docs/contributing/agents/playbooks/update-snapshots.md).
- `tests/**/*` snapshot files used by `pytest-regressions` ‚Äî regenerated via `uv run poe test:regen`.

## Mandatory PR reminders

These are easy to miss when working from an agent and are required by the PR template:

1. **Complete the AI disclosure**. Check "Was generative AI tooling used to co-author this PR?" and fill in the `Generated-by:` trailer with the tool name. Details: [Pull Request Guidelines ¬ß AI-Assisted Contributions](docs/contributing/pull_request.md#ai-assisted-contributions).
2. **Run `uv run poe all` before pushing**. This is the command named in the PR template; it auto-formats then runs the same lint/check/test pipeline as CI. To mirror CI exactly afterwards, run `uv run poe ci` (uses `prek`, does not auto-format).
3. **Fill in "Steps to Test This Pull Request"** with the exact commands you ran locally ‚Äî the maintainers re-run them.
4. **Follow Conventional Commits** ‚Äî the project uses itself to validate commit messages.

## When unsure

- Read the existing tests and user docs to understand the expected behavior before changing code.
- When behavior is ambiguous, assume **backward compatibility with current tests and docs** is required. Add a deprecation window instead of breaking it; see the [deprecate-public-api playbook](docs/contributing/agents/playbooks/deprecate-public-api.md).
- Cross-platform parity matters ‚Äî if you cannot test on macOS or Windows locally, surface that in the PR description.
